# Terminal Appendix 020: `mirror.audit`

Responsible office: Pattern Integrity Working Group.
Related handbook clauses: 7.7, 21.2, 19.0.
Usual observation point: Stairwell B landing three.

## Purpose
`mirror.audit` is recorded here as a low-clearance or compatibility command. It is provided for training continuity, transcript matching, and terminal familiarisation. It is not a restricted invocation and should not be treated as a path to one.

## Sample output
```
Mirror audit complete: one reflection late, none missing.
```

## Common misreadings
- The presence of `mirror.audit` in a transcript does not prove access to a sealed node.
- Sample output may be older than the terminal that produced it.
- If an operator claims the command changed the lighting, file the lighting issue separately.
- If the command returns a queue, do not join the queue unless you can see its end.
