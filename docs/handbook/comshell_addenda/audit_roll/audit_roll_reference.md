# Terminal Appendix 025: `audit.roll`

Responsible office: Maintenance Without Witness.
Related handbook clauses: 10.4, 20.0, 7.7.
Usual observation point: archive alcove 21.2.

## Purpose
`audit.roll` is recorded here as a low-clearance or compatibility command. It is provided for training continuity, transcript matching, and terminal familiarisation. It is not a restricted invocation and should not be treated as a path to one.

## Sample output
```
present, present, absent, present, [blank], present
```

## Handling notes
Repeated use during quiet hours may cause the prompt to appear further away than the keyboard.
When in doubt, print the output, fold it away from yourself, and submit the folded side first.
Do not attempt to reconcile the output with eyewitness reports unless the eyewitness was present before the command was entered.
The handbook recognises that terminals sometimes answer questions that were typed by the room rather than by the operator.

## Common misreadings
- The presence of `audit.roll` in a transcript does not prove access to a sealed node.
- Sample output may be older than the terminal that produced it.
- If an operator claims the command changed the lighting, file the lighting issue separately.
- If the command returns a queue, do not join the queue unless you can see its end.
