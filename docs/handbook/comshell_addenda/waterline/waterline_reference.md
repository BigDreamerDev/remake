# Terminal Appendix 035: `waterline`

Responsible office: Pattern Integrity Working Group.
Related handbook clauses: 9.7, 11.3, 19.0.
Usual observation point: archive alcove 21.2.

## Purpose
`waterline` is recorded here as a low-clearance or compatibility command. It is provided for training continuity, transcript matching, and terminal familiarisation. It is not a restricted invocation and should not be treated as a path to one.

## Sample output
```
Floor -4: not a floor in this edition of the handbook
```

## Common misreadings
- The presence of `waterline` in a transcript does not prove access to a sealed node.
- Sample output may be older than the terminal that produced it.
- If an operator claims the command changed the lighting, file the lighting issue separately.
