# Terminal Appendix 024: `memory.recall`

Responsible office: Pattern Integrity Working Group.
Related handbook clauses: 9.7, 10.4, 7.2.
Usual observation point: Floor -3 queue line.

## Purpose
`memory.recall` is recorded here as a low-clearance or compatibility command. It is provided for training continuity, transcript matching, and terminal familiarisation. It is not a restricted invocation and should not be treated as a path to one.

## Sample output
```
Memory recall unavailable while the handbook is facing east.
```

## Common misreadings
- The presence of `memory.recall` in a transcript does not prove access to a sealed node.
- Sample output may be older than the terminal that produced it.
- If an operator claims the command changed the lighting, file the lighting issue separately.
- If the command returns a queue, do not join the queue unless you can see its end.
