# Codicology Handbook — Contacting the Codicology Wing

**Document:** SW-HB-COD-017  
**Department:** Office of Inherited Silences / Codicology Wing  
**Desk:** Archive Integrity / Returned Box Registry  
**Revision:** 4.1  
**Handling:** Internal. Do not laminate. Do not read beside running water.

---

## 1. Purpose

This section describes the approved method for contacting the Codicology Wing when a record, duplicate record, wet copy, recalled page, returned box, or self-correcting index card requires review.

Codicology does not accept direct verbal requests.

Codicology does not accept knocks.

Codicology does not accept messages written on the reverse side of a page unless the reverse side was blank when you found it.

If the page was not blank when you found it, stop reading and file a silence report.

---

## 2. Approved Contact Route

All requests to Codicology must be placed through comShell.

Open comShell and enter:

```txt
contact.office codicology
```

If the office is available, comShell will return a routing notice.

If the office is unavailable, comShell may return one of the following acceptable responses:

```txt
line present. office absent.
```

```txt
codicology is reviewing the shape of your request.
```

```txt
the archive has received a version of this question already.
```

```txt
the desk is occupied by a chair that was not requisitioned.
```

Do not repeat the command more than three times.

Repeated contact attempts may cause duplicate tickets to appear in your name.

Duplicate tickets are not to be opened.

---

## 3. When to Contact Codicology

Contact Codicology if any of the following occur:

1. A record has more pages after being closed than before being opened.
2. A document remembers an employee who has not yet been hired.
3. A wet signature dries into a different name.
4. An index card lists a room number that is not currently installed.
5. A handbook entry refers to a corridor using affectionate language.
6. A returned box contains only air, but the air is labelled.
7. A page asks to be filed lower.
8. A copied document refuses to match the original because it has “learned better”.
9. A seal has broken inward.
10. A silence report arrives with your handwriting already present.

If the incident involves standing water, mirror glass, stairwell echo, or music heard through a closed drawer, contact Hydrology or Pattern Integrity first.

If both departments refer you back to Codicology, do not mention this to either department.

## 4. Creating a Contact Ticket

To create a contact ticket, enter:

```txt
ticket.open codicology
```

Recommended ticket headings:

```txt
returned box irregularity
```

```txt
wet copy review
```

```txt
duplicate signature conflict
```

```txt
misfiled room memory
```

```txt
index card breathing
```

```txt
record heard before opening
```

Do not use the word “haunted” in a ticket heading.

Use:

```txt
persistent
```

or:

```txt
administratively present
```

## 5. Requesting a Record Pull

If you already know the record family, request an archive pull:

```txt
archive.request codicology
```

Then wait.

Do not type while waiting.

Do not hum while waiting.

If comShell displays a blank line for more than nine seconds, the request is being read by something adjacent to the proper office.

This is normal.

If the blank line begins to indent itself, close the terminal.

This is not normal.

---

## 6. Emergency Contact Procedure

Emergency contact is permitted only when a document has begun altering live infrastructure.

Examples include:

- Door labels changing while observed.
- Room numbers appearing in employee records.
- comShell output continuing after shutdown.
- A handbook section addressing the reader by childhood nickname.
- Archived music playing from a file that is not audio.
- A page becoming wet from the margins inward.

Enter:

```txt
contact.office codicology emergency
```

If prompted for an extension, enter:

```txt
7
```

If the connection routes to hold music, remain on the line until the first complete phrase repeats.

Do not transcribe the melody.

Do not attempt to identify the instrument.

If the music contains structured tones beneath the piano line, log the tones exactly as heard and submit them with the ticket.

Do not solve them in the ticket body.

Codicology dislikes premature interpretation.

---

## 7. Call-Through Behaviour

In some cases, comShell may simulate an internal call.

This is not a telephone call.

It is a request being carried through the archive in the shape of a telephone call.

Expected sequence:

```txt
routing request...
line acquired.
operator unavailable.
transferring to lower switchboard.
hold channel open.
```

If the output includes:

```txt
someone picked up before it rang.
```

close comShell and wait one full minute before reopening it.

If the output includes:

```txt
the receiver is wet.
```

do not continue. Contact Hydrology in person, unless Hydrology is already in the room.

---

## 8. Prohibited Contact Methods

Do not contact Codicology by:

- Writing requests into margins.
- Leaving pages beneath doors.
- Speaking into file boxes.
- Calling numbers found in water damage.
- Asking the cafeteria radio.
- Addressing the locked reading room by name.
- Reading a ticket backwards to see if it answers.
- Sending messages through mirror fog.
- Folding request forms into boats.
- Asking a duplicate employee to “take it down there”.

The Codicology Wing receives all of these methods.

That is why they are prohibited.

---

## 9. Response Windows

Codicology does not follow standard response times.

The following windows are considered administratively acceptable:

| Request Type | Expected Response |
|---|---|
| Returned box | before the box is returned again |
| Wet copy | after drying, before correction |
| Duplicate signature | once both signatories deny it |
| Misfiled room | when the room is next empty |
| Seal break | immediately, retroactively |
| Index card anomaly | when the card is tired |
| Lower switchboard query | after the music stops |

Do not chase a ticket unless comShell explicitly instructs you to remember it.

If comShell instructs you to forget it, record the instruction in a separate document and file it under `forgetting`.


If music is heard, wait until the phrase repeats.

If a second melody answers from beneath it, contact Pattern Integrity after filing the Codicology ticket.

Do not ask which melody was first.

## 11. Notes for Junior Archivists

The Codicology Wing is not lost.

It is indexed elsewhere.

If you cannot find the office, check whether you are holding a document that refers to it in the past tense.

If the document refers to the office in the future tense, leave the document where it is and return after lunch.

If the document refers to the office as “home”, place it face down and inform Archive Integrity.

Do not comfort the document.

---

## 12. Closing Instruction

When finished with a Codicology contact session, enter:

```txt
return.path
```

If comShell responds with a corridor name, do not use that corridor.

If comShell responds with your own name, submit a duplicate signature review.

If comShell responds with silence, wait.

Silence is an accepted form of receipt.

Silence is not an accepted form of permission.
