# Codicology Handbook — Lower Switchboard Audio Retrieval

**Document:** SW-HB-COD-019  
**Department:** Office of Inherited Silences / Codicology Wing  
**Desk:** Archive Integrity / Lower Switchboard  
**Handling:** Internal. Do not perform this procedure near uncovered glass.

---

## 1. Contact Route

Codicology accepts switchboard traffic only when the request arrives in the correct shape.

Open comShell and enter:

```txt
contact.office codicology
```

The terminal may display a lower switchboard panel.

Do not describe the panel in the ticket body.

Do not count the wires.

If the panel asks for an extension, dial the extension assigned to lower archive audio routing.

If you are unsure which extension this is, you are not the intended caller.

---

## 2. Audio Retrieval Shape

Once connected, the switchboard will ask for the record type.

The only accepted type for this route is:

```txt
audio
```

The switchboard will then ask for the audio id.

The lobby hold channel recovered from the lower switchboard is filed as:

```txt
14
```

Any other extension, type, or id will terminate the call.

This is not a security measure.

It is because the other routes answer.

---

## 3. Listening Protocol

When the audio begins, let it loop.

Do not stop playback during the first full repetition.

Do not write down the piano line.

Do not hum the lower tone.

If structured tones are heard beneath the music, record them exactly as heard in a separate note and file the note with Archive Integrity.

Do not solve the tones in the ticket body.

Codicology dislikes premature interpretation.

---

## 4. Failed Call Behaviour

If the call terminates, do not immediately redial.

Wait until the terminal prompt returns.

If the prompt returns with an extra space before the cursor, close comShell and submit a silence report.

If the prompt does not return, the call is still open somewhere else.
